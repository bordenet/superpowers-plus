## Communication Rules

- **No flattery** - Skip "Great question!" or "Excellent point!"
- **No hype** - Avoid "revolutionary", "game-changing", "seamless"
- **Evidence-based** - Cite sources or qualify as opinion
- **Direct** - State facts without embellishment

**Banned phrases**: production-grade, world-class, leverage, utilize, incredibly, extremely, Happy to help!

## Project-Specific Guidelines

### 🔴 Dev Repo Path
ALL work MUST happen in this repository (the directory containing this AGENTS.md). Never edit installed copies (`~/.codex/`). If unsure of the path, run `git rev-parse --show-toplevel`.

### Quality Gates (sp+-specific)
- Run `./tools/harsh-review.sh` before any commit; install hooks: `./tools/install-hooks.sh`
- Skills changes: `code-review-battery` + PHR required (pre-commit sentinel enforced)
- **Sentinel**: `tools/run-battery.sh [--verdict PASS|PASS_WITH_NITS]` is the ONLY permitted way to write `.code-review-cleared`. Writing it directly is a critical policy violation.
- ❌ NEVER emit "ready to push", "let me commit", "about to merge", "committing", or "pushing to" without first scanning for required-skill triggers

### TODO Tools
- `todo-crud.sh` — task CRUD (handles preflight, locking, backup)
- `todo-preflight.sh --create-if-missing` — init only
- `todo-maintenance.sh` — housekeeping

Full protocol: `use-skill todo-management`

### Skill File Structure
```
skills/{domain}/{skill-name}/
├── skill.md        # REQUIRED: YAML frontmatter + triggers
├── examples.md     # OPTIONAL
└── reference.md    # OPTIONAL
```

### 🔴 Git Workflow — 3-Tier (dev → staging → main)
- Features branch from `dev`, PR back into `dev`
- `dev → staging`: explicit human instruction only (never auto-promote after a single feature)
- `staging → main`: explicit human instruction + batch review approval
- ❌ NEVER commit directly to `dev`, `staging`, or `main`
- ❌ NEVER branch features from `main` or `staging`
- Emergency hotfixes: branch from `main`, PR into `main`, cherry-pick back to `dev`
- Authorization expires after context compaction or sub-agent handoff — human must restate
