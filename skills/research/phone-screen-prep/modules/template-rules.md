# Template Output Rules

## 🚨🚨🚨 THE TEMPLATE IS THE OUTPUT — DO NOT STRIP IT 🚨🚨🚨

<EXTREMELY_IMPORTANT>

The file at `templates/phone-screen.md` defines the **complete structure** of every phone screen prep file.

**REQUIRED SECTIONS (NEVER REMOVE):**

| Section | Purpose | Lines |
|---------|---------|-------|
| ⚠️ **Integrity Policy Callout** | Verbatim script to read at start of call | Top |
| **PRE-SCREEN ASSESSMENT** | Signal table + Four Quadrants + Prioritized Questions | ~30-60 |
| **Running Notes** | Raw observations and candidate questions during call | ~65-75 |
| **25-Min Call Flow** | Complete scripted structure for the call | ~75-180 |
| ├─ 0-5 min: Rapport + Context | Intro script + integrity policy checkbox | |
| ├─ 5-20 min: Questions | Q1-Q4 blocks with notes sections | |
| ├─ 20-25 min: Sell + Q&A | Selling points, comp confirmation, their questions | |
| ├─ **Comp Confirmation** | Base-only script + alignment table | |
| └─ **If Advancing to Loop** | Mandatory script + checkboxes | |
| **Post-Call Eval** | Four Quadrants checklist + Decision | ~185-215 |
| **Actions** | Checkboxes for next steps | ~220-230 |

### What Happens When You Strip Sections

**Ryuta's file (WRONG):** Missing call flow, integrity policy, comp confirmation, loop script
→ Matt has no guidance during the call, must wing it

**Bruce's file (CORRECT):** Full template, all scripts, all checkboxes
→ Matt follows the runbook during the call

### The Rule

1. **READ the template first** — `view templates/phone-screen.md`
2. **COPY IT WHOLESALE** — Every section, every script, every checkbox
3. **FILL IN the blanks** — Replace `[placeholders]` with candidate info
4. **ADD candidate-specific sections** — Agency callout, known-candidate callout, etc.
5. **NEVER DELETE template sections** — Only add, never subtract

**If a section doesn't apply (e.g., comp gate for agency candidate), keep the section header and note "N/A — agency-sourced"**

</EXTREMELY_IMPORTANT>
